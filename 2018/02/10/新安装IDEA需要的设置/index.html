新安装IDEA需要的设置

Posted on 2018-02-10 | In 实用工具

由于我比较喜欢折腾，喜欢最新版的软件，最新版的系统，所以windows版本比较喜欢用预览版，所以这就导致了总是会遇到bug，可能就需要重装系统了，装系统倒是不麻烦，上面的软件才是最麻烦的，所以此文用来记录每次IDEA装好之后需要进行的配置，保证每次装好之后环境都一样的，

设置工作环境的编码（默认都是utf-8)

针对各种乱码的终极解决办法

  • 进入idea的安装目录, 进入bin目录下。找到idea.exe.vmoptions这个文件 和 idea64.exe.vmoptions，针对自己安装的版本修改：用记事本打开末尾添加如下字符
  • -Dfile.encoding=UTF-8

全局编码设置

  • File –> Other Settings –> Default Settings
  • Edit –>File Encodings (打开了这个页面，也不用我多说了，看到需要设置的都改了)

针对本项目的编码设置

  • File –> Settings –> Default Settings
  • Edit –>File Encodings (打开了这个页面，也不用我多说了，看到需要设置的都改了)

字体设置

  • Editor –> Colors & Fonts –> Font(这个页面都能看到所有设置)
  • 字体喜好：Consolas

Maven设置

  • File –> Other Settings –> Default Settings
  • Build, Execution, Deployment –>Build Tools –> Maven(Maven各个配置)

JDK设置

  • File –> Other Settings –> Default Project Structure
  • Project选择安装好的jdk版本

取消编码过程中的大小写区分

  • File –> Other Settings –> Default Settings
  • Editor –> General –> Code Completion –>Case sensitive completion –>None (设置完成)
